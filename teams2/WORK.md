# WORK Team 2 | SD-WAN Simulation Lab

## Архитектурный ландшафт проекта

данный проект моделирует поведение интеллектуального узла связи, который выбирает маршрут не просто по доступности интерфейса, а на основе реального качества канала (RTT и потери пакетов)

---

## Глоссарий компонентов

### Инфраструктурный слой
*   **Dockerfile**
    собирает боевое окружение на базе легкого образа, упаковывая внутрь сетевой стек iproute2, инструменты диагностики ping/tc и интерпретатор python 3
*   **docker-compose.yml**
    дирижер оркестрации, который разворачивает два независимых филиала и связывает их двумя изолированными сетями (fiber для оптики и satellite для спутника), выдавая контейнерам привилегии NET_ADMIN для прямого управления ядром
*   **branch_entrypoint.sh**
    стартовый скрипт-инициализатор, который размечает ip-адреса, задает начальные приоритеты маршрутов и активирует эмуляцию деградации (netem) на спутниковом канале

### Программный интеллект (Python)
*   **collector.py**
    глаза системы: выполняет серию icmp-запросов, жестко привязанных к конкретным интерфейсам, и вычленяет из сырого вывода точность до миллисекунды
*   **engine.py**
    мозг системы: сопоставляет полученные данные с пороговыми значениями и использует механизм гистерезиса (cooldown), чтобы избежать «болтанки» маршрутов при нестабильном сигнале
*   **net_manager.py**
    руки системы: безопасно взаимодействует с таблицей маршрутизации linux, подменяя путь через нужный шлюз одной атомарной операцией
*   **main.py**
    сердце системы: бесконечный цикл, который координирует работу всех модулей и генерирует информативные логи для демонстрации

## Инструкция по развертыванию

### Подготовка
убедитесь, что в вашей системе установлен docker и демон запущен

### Запуск стенда
выполните команду сборки и запуска в терминале, находясь в корне проекта
```bash
docker compose -f team2/docker-compose.yml up --build
```

### Наблюдение за логикой
откройте новое окно терминала для мониторинга в реальном времени того, как «думает» контроллер
```bash
docker logs -f sdwan_branch_a
```

### Сценарии для живой демонстрации
> Имитация аварии на магистрали
выключите интерфейс оптического канала в активном филиале
```bash
docker exec -it sdwan_branch_a ip link set eth1 down
```
система зафиксирует 100% потерь на fiber и мгновенно перестроит таблицу маршрутизации на satellite

### Восстановление и возврат
верните оптику в строй и наблюдайте за автоматическим возвратом трафика
```bash
docker exec -it sdwan_branch_a ip link set eth1 up
```
благодаря настройке cooldown, возврат на приоритетный канал произойдет плавно через 15 секунд после его стабилизации

### Режим демонтажа
чтобы полностью очистить систему от созданных сетей и контейнеров, используйте команду
```bash
docker compose -f team2/docker-compose.yml down
```

## Итог
демонстрация доказывает, что в отличие от классического роутинга, наш sd-wan узел способен поддерживать связность филиалов даже при деградации физических линий, обеспечивая бесшовный пользовательский опыт за счет автоматизации управления путями передачи данных
